## Architecture Rules

1. Follow Clean Architecture.
2. Follow SOLID Principles.
3. Use Modular Design.
4. Keep Business Logic Separate.
5. Use Reusable Components.

---

## Security Rules

1. Use JWT Authentication.
2. Hash Passwords.
3. Validate All Inputs.
4. Prevent SQL Injection.
5. Prevent Cross Site Scripting (XSS).
6. Implement Role Based Access Control.
7. Maintain Audit Logs.

---

## Coding Standards

1. Use Meaningful Names.
2. Write Comments for Critical Logic.
3. Avoid Duplicate Code.
4. Create Reusable Services.
5. Follow TypeScript Best Practices.

---

## Database Rules

1. Use Primary Keys.
2. Use Foreign Keys.
3. Maintain Referential Integrity.
4. Store Audit Information.
5. Never Physically Delete Important Records.
6. Use Soft Delete Where Required.

1. Maintain Complete Audit Trail.

## AI Development Instructions

Before writing code:

1. Read all project markdown files.
2. Review dependencies.
3. Identify missing requirements.
4. Ask questions if anything is unclear.
5. Generate implementation plan.
6. Build one module at a time.
7. Update TASKS.md after every completed module.
